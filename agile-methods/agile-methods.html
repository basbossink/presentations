    What is agile?

    Agile software development
    a group of software development methods based on iterative and incremental development, where requirements and solutions evolve through collaboration between self-organizing, cross-functional teams.
    Wikipedia: Agile Software Development

    Individuals and interactions
    over processes and tools
    Working software
    over comprehensive documentation
    Customer collaboration
    over contract negotiation
    Responding to change

    Lean Software Development

    • book by Mary and Tom Poppendieck
    • influenced by:
      • Toyata Production System
      • Lean manufacturing

    Lean Software Development

    Seven Principles

    1. Eliminate waste
    2. Amplify learning
    3. Decide as late as possible
    4. Deliver as fast as possible
    5. Empower the team
    6. Build integrity in
    7. See the whole

    Lean Software Development

    Seven Practices

    1. Seeing waste
    2. Value stream mapping
    3. Set-based development
    4. Pull systems
    5. Queing theory
    6. Motivation
    7. Measurements

    Kanban

    Kanban

    • literally signboard or billboard
    • influenced by:
      • Toyata Production System
      • Lean Manufacturing
      • Lean Software Development

    Kanban

    Four principles

    1. Start with what you do now (don't change anything)
    2. Agree to pursue incremental, evolutionary change
    3. Respect the current process, roles, responsibilities & titles
    4. Leadership at all levels

    Kanban

    Six Practices

    1. Visualise
    2. Limit WIP (work-in-progress)
    3. Manage flow
    4. Make policies explicit
    5. Implement feedback loops
    6. Improve collaboratively, evolve experimentally
